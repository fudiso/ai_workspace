# AI WorkSpace 추진 보고서
**AI전략센터**

## 1. 추진 개요
전사 임직원이 AI의 도움을 받아 데이터를 자유자재로 활용하는 'Data-Native' 조직으로의 전환을 목표로 함. 이를 위해 데이터 접근성을 극대화하고, 직무 및 역량 수준에 맞는 맞춤형 분석환경을 제공하고자 함. 

## 2. 프로젝트 목적
모든 임직원들이 데이터에 자유롭게 접근하고, 직관적인 인터페이스와 AI 기술을 활용하여 스스로 데이터를 찾아 분석하며, 반복적인 업무를 자동화할 수 있는 통합 환경을 구축하는 것을 목적으로 함. 이를 통해 데이터 기반의 신속하고 정교한 의사결정을 지원하고, 전사적인 업무 효율성과 생산성을 극대화하고자 함.

## 3. 현황 및 문제점
우리금융지주 각 자회사(은행/카드/증권/생명보험/캐피탈/자산운용)의 업무 담당자들은 데이터 기반 업무를 수행하기 위해 '데이터 저장 → 데이터 가져오기 → 처리하기 → 의사결정하기'의 흐름을 거치지만, 이 과정에서 여러 구조적 문제에 직면하고 있음.

데이터 분석 요청이 IT 부서에 집중되어 업무 지연이 발생하고, 개별 업무 담당자의 노하우가 부서별/개인별로 분산되어 시스템화하기 어려운 상황임. 대부분의 업무 담당자는 Excel 기반의 수작업에 의존하거나, 외부 정보단말기(블룸버그, Factset 등)에 의존하고 있어 내부 데이터 자산을 충분히 활용하지 못하고 있음.

특히 Python 등 분석 도구에 대한 기술적 진입장벽과 실행 환경의 부재로 인해, 업무에 필요한 데이터 처리와 분석이 개인 PC의 Excel 파일 수준에 머물러 있어 조직 차원의 자산 축적이 이루어지지 않고 있음.

### 문제점 요약

| 문제 영역 | 주요 이슈 요약 |
|-----------|----------------|
| IT 개발 인력 의존성 | 모든 업무담당자에게 IT개발직원을 배치하는 것은 현실적으로 불가능하며 비용이 과다 |
| 업무 프로세스 시스템화 한계 | 개인/부서별 노하우가 다양하여 시스템화하기 어려운 업무가 대부분 |
| 코딩 환경 부재 | 코드 실행 환경과 robust한 DB 시스템이 없어 AI 코딩 활용 불가 |
| 데이터 자산 분산 | 정형/반정형/비정형 데이터가 체계적으로 관리되지 않아 접근성 저하 |
| 개인 노하우의 조직 자산화 실패 | Excel 기반 개인 업무는 회사 차원의 자산으로 축적되지 못함 |
| 외부 단말기 의존성 | IT부서 지원 부족 시 고비용 외부 정보단말기에 의존할 수밖에 없는 구조 |
| 코딩 역량 부족 | 생성형 AI 시대에도 코딩을 업무 도구로 활용할 수 있는 환경과 교육 부재 |

## 4. AI WorkSpace(가칭) 솔루션
"데이터를 찾기 어렵고, 분석 역량은 제한적이고, 노하우가 축적되지 못하는" 구조적 문제를 해결하기 위해, 데이터 민주화 플랫폼을 지향하는 AI WorkSpace 프로젝트를 추진하고자 함. AI WorkSpace는 정형 및 반정형데이터의 탐색과 활용에 중점을 두며, Microsoft Copilot이 담당할 비정형 문서 데이터 영역과 함께 전사적인 AI기반 데이터 활용 생태계를 완성하고자 함.

필요한 데이터를 사내에서 찾지 못해 외부 정보 단말기에 의존하던 문제를 해결하기 위해 AI WorkSpace는 강력한 '데이터 디스커버리 플랫폼(Data Discovery Platform)'의 역할을 수행. AI의 도움으로 내부 데이터를 쉽게 찾아 활용하는 새로운 업무방식을 제시하며, 그 핵심 구성요소와 작동 방식은 다음과 같음.

**[그림1: AI WorkSpace 핵심 구성 요소]**

### 4.1 핵심 구성요소

#### 1) 데이터 카탈로그 (Data Catalog)
- 전사에 공유 가능한 데이터를 한곳에 모아 데이터 사일로를 해소하고, 사용자가 필요한 데이터를 탐색할 수 있는 기반을 마련
- 이를 통해 외부 단말기에서 데이터를 다운로드하는 번거로움을 줄이고, 검증된 내부 데이터 활용 촉진

#### 2) 코드 카탈로그 (Code Catalog)
- 업무 프로세스와 데이터 분석 노하우를 코드로 자산화하여 체계적으로 축적하고 관리함
- 부서별 유사업무의 중복 수행 문제를 해결하고 분석의 효율성과 일관성을 높임

#### 3) LLM기반 AI 어시스턴트
- 자연어로 질의하면 AI어시스턴트가 Python코드나 SQL쿼리로 자동 변환하여 데이터를 조회
- 코드에 대한 기술적 장벽을 낮추는 역할을 함

#### 4) 클라우드 IDE 및 실행환경
- 웹 브라우저를 통해 바로 접속할 수 있는 코드 실행 및 개발환경
- AI어시스턴트를 통해 생성했거나 코드 카탈로그에서 가져온 코드를 바로 실행 가능
- 개인이 직접 개발환경을 구성할 필요가 없음

#### 5) 워크플로우 자동화
- 코드 카탈로그에 등록된 코드를 워크플로우 자동화 툴과 연계하여 반복적으로 수행가능

## 5. 구축 전략
AI WorkSpace는 오픈소스 기반의 맞춤형 구축을 통해 비용 효율성과 기술적 유연성을 확보하면서, 전 구성원이 '코딩'을 업무 도구로 활용할 수 있는 High-Code 환경 조성을 목표로 하는 전략을 설정.

### 5.1 High-Code 지향 오픈소스 기반 구축
- 상용 AI플랫폼(Palantir, Dataiku 등) 대신 생성형 AI를 활용한 코딩 중심 접근법으로 전환
- No-code/Low-code 도구도 결국 러닝커브가 존재하므로, 차라리 '끝판왕' 코딩을 배워 범용성 확보
- 전문 기술업체 및 웹 개발 파트너와 협업하여 오픈소스 플랫폼 기반 환경 구축
- Cursor, Claude Code, Gemini CLI 또는 Qwen-Code 온프레미스 활용으로 LLM 기반 코딩 환경 제공

### 5.2 핵심 구성 요소별 전략
- **Robust한 데이터 인프라**: 기간계/정보계와 별도로 잘못된 요청에도 견딜 수 있는 구조. Snowflake 기반 데이터 통합 인프라 구축
- **데이터 카탈로그**: 정형/반정형/비정형 데이터를 체계적으로 관리. AI Code Assistant가 메타정보를 바탕으로 쿼리 작성 가능하도록 설계
- **코드 카탈로그**: 다양한 샘플 코드를 저장하여 그대로 사용하거나 변형 가능. GitLab 기반 버전 관리
- **AI Code Assistant 환경**: Cursor, Claude Code, Gemini CLI 또는 Qwen-Code 온프레미스 도입으로 생성형 AI 코딩 지원
- **클라우드 코드 실행 환경**: 웹 브라우저로 접속 가능한 IDE 환경. VS Code Server, JupyterHub 활용
- **워크플로우 자동화**: n8n 또는 Apache Airflow를 통해 반복 업무 자동화
- **웹 포탈**: 단계별 사용자를 위한 통합 인터페이스 제공

### 5.3 사용자 단계별 접근 전략 및 AI 코딩 도입

**사용자 그룹 단계별 진화 경로:**
- **일반 사용자(User)**: 자연어 질의를 통한 데이터 조회 및 기본 분석 기능 사용
- **고급 사용자(Power User)**: 코드 카탈로그에서 기존 코드를 가져와 변형/활용하여 맞춤형 분석 수행
- **데이터 분석가(Data Analyst)**: AI Code Assistant의 도움을 받아 새로운 분석 코드 작성 및 자산화
- **데이터 전문가(Data Scientist)**: 복잡한 알고리즘 및 모델 개발, 전사 분석 표준 수립

**High-Code 지향 전략:**
- No-code/Low-code 도구 대신 생성형 AI를 활용한 직접적 코딩 능력 배양
- 엔드유저가 '코딩'을 Excel처럼 자연스럽게 사용할 수 있는 환경 조성
- 러닝커브를 최소화하고 실질적 업무 효과를 단기간에 체감할 수 있는 교육 및 지원 체계

**조직 역할 재정의:**
- **AI전략센터**: 플랫폼 운영, 금융 특화 AI 모델 큐레이션, 자회사 AI 코딩 도입 가이드
- **IT부서**: 데이터 인프라 운영 및 재사용 가능한 코드 컴포넌트 제공
- **각 자회사 현업**: 주도적 데이터 분석 수행 및 업무 프로세스 자동화

### 5.4 지속 가능한 운영 및 거버넌스
- 고품질의 데이터, 분석 코드, 자동화 플로우를 중앙 저장소에 등록하고 공유함으로써 조직의 분석 자산을 체계적으로 관리. 자산 품질관리를 위한 체계 필요
- 단계별 캠페인, 핵심사용자 발굴, 부서장 대상 워크숍등 커뮤니케이션 전략을 통해 사내 인식 전환 및 지지 기반 확보
- 플랫폼 효과를 정기적으로 경영진에 리포팅하고, 전사 전략과 연계된 성과를 통해 지속적 지원 확보

## 참고 2. 사용자 시나리오

### 시나리오 1: [우리은행] 상품 포트폴리오 성과 분석 - AI 코드 어시스턴트 활용
1. **(문제 상황)** 우리은행 상품기획팀 김O수 대리는 매일 아침 여러 시스템을 통해 펀드랩, WM상품, 신탁상품의 성과 데이터를 수집하여 일일 보고서를 작성하는데 30분-1시간 소요됨.
2. **(데이터 질의)** AI Code Assistant에게 "우리은행 판매 중인 펀드랩 상품들의 최근 1개월 수익률과 자금 유출입 현황을 분석해줘"라고 요청. 데이터 카탈로그의 메타정보를 참조하여 필요한 SQL 쿼리와 Python 코드를 자동 생성.
3. **(코드 수정 및 실행)** 웹 브라우저를 통해 클라우드 IDE에 접속하여 생성된 코드를 실행. AI Code Assistant에게 "벤치마크 지수와의 상대성과 분석도 추가해줘"라고 요청하여 코드 개선
4. **(코드 자산화)** 검증된 분석 코드를 "은행 상품 포트폴리오 일일 성과분석"로 코드 카탈로그에 등록하여 전사 자산화. 상품기획팀, WM영업본부 등에서 재사용 가능
5. **(반복 업무 자동화)** 검증된 코드를 워크플로우 자동화 도구(n8n 등)에 등록하여 매일 오전 8시 자동 실행. 분석 결과를 Excel로 출력하고 Teams/이메일로 관련자에게 자동 전송

### 시나리오 2: [우리카드] 고객 세그먼트 분석 - 코드 카탈로그 활용
1. **(문제 상황)** 우리카드 상품마케팅팀 박O영 과장은 고객의 신용등급, 소비패턴, 연령대별로 최적의 카드 상품을 추천하는 개인화 마케팅 전략을 수립해야 하는데, 기존 방식으로는 세그먼트 분석에 많은 시간이 소요됨.
2. **(데이터 및 로직 확보)** 박 과장은 AI WorkSpace 포털에 접속하여 **'코드 카탈로그'**에서 '고객 세그먼트 분석' 코드를 불러옴. 이 코드는 카드 이용 데이터, 고객 정보 DB에서 고객별 특성 정보를 가져오는 기능을 포함하고 있음.
3. **(코드 수정 및 분석)** 박 과장은 AI Code Assistant에게 "20-30대 직장인 중 온라인 쇼핑 이용률이 높은 고객군을 추출하고, 이들에게 적합한 카드 상품을 매칭해줘"와 같은 자연어 질의를 통해 코드를 수정
4. **(결과 확인 및 저장)** 클라우드 IDE 실행 환경에서 AI에 의해 수정된 코드를 수행후 결과 확인. 고객 세그먼트별 상품 추천 로직이 검증되면 '연령대별 맞춤형 카드 추천 시스템'이름으로 '코드 카탈로그'에 저장하여 자산화

### 시나리오 3: [우리증권] 시장 급변 시 포트폴리오 리스크 분석
1. **(이슈 발생)** '미 연준, 금리 0.5% 전격 인상'이라는 뉴스를 보고 우리증권 리스크관리팀 최O진 차장은 금리 변동에 민감한 고객 포트폴리오를 즉시 파악하고자 함.
2. **(자연어 질의)** 최 차장은 AI WorkSpace LLM 채팅창에 "현재 우리증권 고객들의 포트폴리오 중에서, 듀레이션이 5년 이상인 채권 비중이 30% 이상인 계좌 리스트와 예상 손실률을 알려줘"라고 입력.
3. **(결과 분석)** 시스템은 증권 계좌 정보 DB와 보유 종목 DB를 동시에 조회하여 해당 고객 계좌 목록과 리스크 지표를 즉시 테이블 형태로 제공.
4. **(자동 대응)** 최 차장은 분석 결과를 바탕으로 "고위험 계좌 담당 PB들에게 즉시 Teams로 알림 메시지를 보내고, 고객 상담 스크립트를 자동 생성해줘"라고 지시하여 신속한 고객 대응을 수행

### 시나리오 4: [우리생명] 보험상품 추천 콘텐츠 자동화
1. **(콘텐츠 작성)** 우리생명 상품기획팀 이O수 팀장은 '2025년 연금보험 상품 가이드' 영상 콘텐츠를 제작하고 홈페이지와 모바일앱에 게시하고자 함. 고객 상담센터 AI챗봇에서도 해당 콘텐츠를 효과적으로 활용하기 위해 동영상 스크립트와 상품 정보를 구조화해야 했음.
2. **(자동화 실행)** 이 팀장은 AI WorkSpace 포털에서 "보험상품 콘텐츠 등록 자동화" 워크플로우를 실행하고, 영상 URL과 상품코드 정보를 입력.
3. **(프로세스 자동화)** 워크플로우는 입력된 URL에서 영상 제목, 설명, 자막 정보를 자동 추출하고, 상품 DB에서 해당 보험상품의 상세 정보를 가져와 LLM으로 고객 FAQ 형태로 변환, 구조화된 정보를 상담 시스템 DB에 저장
4. **(AI상담 연동)** 고객이 모바일앱에서 '연금보험 추천'이라고 문의하면 AI챗봇이 새롭게 등록된 영상 콘텐츠와 함께 맞춤형 상품 정보를 제공

### 시나리오 5: [우리금융지주 IT-현업 협업] 그룹 통합 리포팅 시스템 구축
1. **(IT 부서의 역할 변화)** 우리금융지주 IT부서는 기존 그룹사 실적 집계 시스템의 '재무 분석' 기능을 각 계열사 현업에서 직접 활용할 수 있도록 '코드 카탈로그'에 패키지 형태로 제공. 이 패키지에는 '그룹 재무분석 함수'와 필요한 '데이터 명세(계열사 코드, 기준월 등)'가 함께 포함되어 있음.
2. **(현업의 주도적 분석)** 우리은행 기획팀 담당자는 이 코드를 자신의 분석 환경으로 가져와, 특정 기간과 사업부문을 입력하여 직접 은행 부문 성과 분석을 수행.
3. **(자체적 확장)** 더 나아가, 담당자는 AI Code Assistant의 도움을 받아 기존 코드에 '디지털 채널별 성과 기여도'를 추가로 분석하는 로직을 덧붙여 자신만의 분석 툴로 발전. IT 부서에 별도의 개발을 요청하고 기다릴 필요 없이, 현업이 직접 필요한 기능을 즉시 구현하고 활용
4. **(그룹사 공유)** 수정한 로직을 그룹 전체에서 활용할 수 있도록 '코드 카탈로그'에 업로드하여 우리카드, 우리증권, 우리생명에서도 활용 가능
